# (My Shell) US002 - Non-Functional Requirements

---

## 1. Introduction

The project development of any professional software product requires a set of non-functional requirements to be followed by the development team.

---

## 2. Requirements

- A list of non-functional requirements must be defined in a form that is professional and understandable by the development team.

### 2.2 Acceptance Criteria

1. A considerable number of non-functional requirements must be defined, written in a professional and understandable manner for the development team.

### 2.3 Dependencies/References

No dependencies.
